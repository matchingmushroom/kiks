Bug Fixes

* Fix the bug where coupons incorrectly display an Expired error message when applied.
* Fix the issue where the WhatsApp redirect feature fails to work on mobile devices.
* Resolve the failure preventing custom messages with coupon codes from working or sending correctly.

Order and Sales Management

* When a user clicks the Sale from Order button within the order details screen, all available information from that order must automatically populate the new sale form.
* Provide users with the native option to amend existing orders.
* Automatically generate an invoice based on the unique Sales ID immediately after a sale is saved.
* When a sale is completed, arrange the workflow to automatically remove the corresponding orders from the order tab.

Coupon Integration

* Add a new column in the Record Sale section that allows users to either assign or automatically generate a personalized coupon valid for 1 month. This coupon must be mentioned on the generated invoice, and the system must simultaneously create this coupon in the main Coupons section.

Search and UI Navigation

* Add a search bar with a slicer-style filter to the Sales tab. Clicking an item in the resulting search box or table should open that specific sales detail page in View-Only mode.
* Clicking on a client name on the Home page should open a view filtered strictly to that specific customer's details.
* In the Invoices tab, create sub-tabs to clearly segregate Invoices and Estimates.

Suppliers and Customers Modules

* Add a Suppliers module to list and record basic supplier details including Name, Location, Phone, Website, Key Contact Person Name, and their Mobile Number. This data should be used to populate dropdown menus wherever supplier information is required.
* Create and integrate a Customer List module that functions in the same manner as the Suppliers module.

Purchasing and Financials

* Update the Payment Method selection in the Add Purchase form to include a Credit option.
* Create a separate Creditor Module similar to the existing debtor module, and ensure this module is properly reflected within the Financial tab and its associated reports.

Permissions and Access Control

* Update permissions to allow Staff roles to generate invoices.
* Remove access to the Setup and Settings tabs for the Manager role.
* Restrict the Staff Dashboard view so staff members cannot see high-level data meant for Admin, Manager, or Accountant roles. It should only display charts and graphs based strictly on the sales recorded by that particular logged-in user.
* Create an Access Control tab that lists every system module and function. The admin must be able to use this tab to grant or restrict access to specified users by defining which modules and functions are allowed when creating or updating a user role.
